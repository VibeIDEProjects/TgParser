# TgParser — Продуктовая спецификация (PRODUCT.md)

<!-- vibeVersion: 1.0.0 -->

## 1. Назначение продукта

TgParser — консольная утилита для извлечения сообщений из Telegram-каналов (открытых и закрытых) с сохранением в структурированные форматы (JSON, CSV, текст).

**Пользователь:** разработчик, аналитик, исследователь — тот, кому нужен доступ к содержимому каналов вне клиента Telegram.

## 2. Пользовательские сценарии

### 2.1. Парсинг открытого канала
1. Пользователь выполняет `tgparser parse open @channel_name`
2. Утилита авторизуется через MTProto (Telethon), используя сохранённую сессию
3. Извлекает N последних сообщений (по умолчанию 100)
4. Выводит результат в stdout в выбранном формате

### 2.2. Парсинг закрытого канала
1. Пользователь выполняет `tgparser parse closed https://t.me/+abc123`
2. Утилита открывает headful-браузер с web-версией Telegram
3. Авторизуется через QR-код (если сессия отсутствует/протухла)
4. Переходит по ссылке канала
5. Скроллит и извлекает сообщения из HTML DOM
6. Обходит защиту от копирования (если активна)
7. Выводит результат

### 2.3. Первичная авторизация
1. Пользователь выполняет `tgparser auth`
2. Открывается браузер с `web.telegram.org`
3. Генерируется QR-код
4. Пользователь сканирует QR через телефон (Telegram → Настройки → Устройства → Подключить устройство)
5. Сессия сохраняется локально
6. Последующие запуски используют сохранённую сессию

## 4. Инварианты

- **Консистентность:** извлечённый текст сообщения совпадает с видимым в клиенте (с точностью до эмодзи/форматирования).
- **Идемпотентность:** повторный парсинг одного канала с теми же параметрами даёт одинаковый результат (при отсутствии новых сообщений).
- **Безопасность сессии:** сессионные данные не покидают локальную машину пользователя.
- **Не хуже ручного копирования:** автоматический обход защиты от копирования извлекает не меньше текста, чем пользователь может скопировать вручную.

## 5. Edge-cases

| Ситуация | Поведение |
|----------|-----------|
| QR-код протух (истёк таймаут) | Перегенерировать QR, показать новый |
| Сессия протухла | Потребовать повторную авторизацию |
| Канал не найден / нет доступа | Сообщение об ошибке, exit code ≠ 0 |
| Rate-limit (Too Many Requests) | Exponential backoff, повтор через N секунд |
| Канал содержит только медиа без текста | Сохранить ссылку на медиа, текст — null |
| Длинное сообщение (>4096 символов) | Извлечь полностью |
| Закрытый канал сменил invite-ссылку | Сообщение об ошибке |
| Telegram Web изменил вёрстку | Логгирование, fallback-селекторы |
| Пользователь закрыл браузер во время авторизации | Корректный выход с сообщением |

## 6. Не-функциональные требования

- **Производительность:** 100 сообщений открытого канала — не более 10 секунд
- **Надёжность:** 3 retry при transient-ошибках (сеть, rate-limit)
- **Приватность:** никакие данные не отправляются на внешние серверы (кроме API Telegram)

## 7. Критерии успеха

- [ ] Утилита собирается и запускается одной командой (`pip install . && tgparser ...`)
- [ ] Успешный парсинг 100 сообщений из открытого канала менее чем за 10 секунд
- [ ] Успешный парсинг 50 сообщений из закрытого канала (через web) с обходом запрета копирования
- [ ] Авторизация через QR работает с первой попытки
- [ ] Повторный запуск без переавторизации
